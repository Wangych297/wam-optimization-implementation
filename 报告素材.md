# 报告素材

## 项目定位

本任务围绕信息安全课程大作业中的数字水印方向展开，重点关注鲁棒图像水印、版权保护、内容溯源、篡改/裁剪/拼接后的水印检测与恢复。

当前优先考虑的技术路线是：

> 以 WAM（Watermark Anything with Localized Messages）作为主论文候选，以 DWSF（Practical Deep Dispersed Watermarking with Synchronization and Fusion）的分散嵌入、同步定位、消息融合思想作为主要改进来源。

## 当前阶段结论

WAM 目前比 DWSF 更适合作为主论文候选，原因是 WAM 官方代码和预训练权重更明确，能在本地先跑通推理和攻击评测；DWSF 思想很适合改进 WAM，但 DWSF 本体训练依赖 A100 且缺少现成预训练权重，在 4060 上直接复现主结果风险较高。

当前主论文候选优先级：

1. WAM / Watermark Anything with Localized Messages
2. EditGuard
3. TrustMark
4. CRMark
5. Robust-Wide / VINE

当前不建议作为主论文的对象：

- PIMoG / RoPaSS：物理拍屏依赖强，复现实验不稳定。
- DWSF：适合作改进思想来源，但直接作主论文复现风险高。
- RAIMark / Ultra-high Resolution：方向贴合但代码和训练资源风险较大。
- CIN / FIN / MuST / SepMark：可作为参考或基线思想，但直接大改风险偏高。

## WAM 为什么适合作主论文

WAM 的核心任务是局部化图像水印。它可以在图像局部区域嵌入消息，并在图像经历裁剪、拼接、局部编辑或 inpainting 后，检测哪些区域含有水印并恢复对应消息。

它适合作主论文的理由：

- 方向属于数字水印、版权保护、内容溯源，符合信息安全课程范围。
- 官方代码和预训练权重公开，先跑通推理和实验的概率较高。
- 支持局部水印和多消息，与 DWSF 的多块分散嵌入、消息融合思想天然相近。
- 实验容易设计：可比较单区域嵌入、多区域嵌入、多区域多数投票、多区域置信融合等方案。

主要风险：

- 如果只调用 WAM 并加一个简单投票，会显得创新不足。
- 需要把 DWSF 式分散冗余嵌入和融合恢复设计成完整实验，而不是简单后处理。
- WAM 本身不是专门解决几何同步校正的论文，因此不要硬吹完整几何鲁棒，应重点放在裁剪、拼接、局部破坏、多区域融合恢复。

## DWSF 可借鉴思想

DWSF 的主要可迁移思想：

- 分散嵌入：在多个区域重复嵌入同一消息，提高裁剪和局部破坏后的存活率。
- 同步定位：先定位水印区域，再进行解码，缓解攻击后水印块位置变化的问题。
- 消息融合：多个区域分别解码后，根据相似度、置信度或一致性融合出最终消息。

拟用于 WAM 的改进方向：

- 在同一张图中选择多个局部区域嵌入同一消息。
- 攻击后利用 WAM 的 detection mask 找到水印候选区域。
- 对多个区域输出的 bit 结果进行 majority vote、confidence fusion 或 DWSF-style similarity fusion。
- 针对 crop、splicing、local occlusion、inpainting、JPEG、resize 等攻击评估 bit accuracy 和 detection quality。
