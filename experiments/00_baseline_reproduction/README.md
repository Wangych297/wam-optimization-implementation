# 00 基础复现

## 目标

本实验用于确认工程的基础水印流程可运行，包括图像读取、消息生成、水印嵌入、水印提取、指标统计和可视化保存。它本身不是主要创新模块，而是后续所有改进的共同参照。

基础复现的意义在于把模型能力固定下来：如果后续模块在压缩、裁剪、局部删除或多源合成场景中效果变化，可以直接和这里的原始提取结果对比，判断改动到底提升了鲁棒性，还是只是改变了输入和统计方式。

## 工程化改动

本目录把基础流程封装成独立实验入口，运行时统一读取 `checkpoints/`、`configs/` 和 `assets/`，输出统一写入 `results_output/baseline_reproduction/`。这样后续模块不需要重复写权重加载和路径处理逻辑。

基础复现实验会保存两类结果：

- 指标文件：记录每张图的提取正确率、消息匹配情况和运行参数。
- 可视化文件：保存原图、水印图和中间结果，便于人工检查水印是否带来明显视觉损伤。

## 与创新模块的关系

后续创新都以这里为起点展开：

- 空间冗余模块会把单次嵌入扩展为多区域嵌入。
- 鲁棒性配置模块会改变嵌入强度并比较画质和恢复率。
- 平台变换模块会在基础水印图上叠加真实传播中的压缩和缩放。
- 篡改定位和来源追踪会把“整图是否能提取”扩展为“局部区域能否解释来源和完整性”。

因此本实验的价值是建立一条干净的主流程，并让后面的每个创新都有可比较的基线。
