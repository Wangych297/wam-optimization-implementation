# 04 平台变换评估

## 目标

本实验模拟图像在平台上传、预览、转发、再保存过程中可能经历的变换。单独测试压缩或缩放只能覆盖一部分情况，实际平台往往会把多种处理串在一起，例如先缩放再压缩，或者格式转换后再次导出。

## 创新设计

本模块把攻击从“单点破坏”扩展为“传播链路”。实验不只问某一种攻击能不能扛住，而是问水印在更接近实际应用的连续变换后还能否恢复。

平台变换评估主要做三件事：

- 组合变换：把压缩、缩放、格式转换等操作按固定模式组合。
- 模式分档：将结果整理成质量优先、均衡鲁棒、安全优先等可选择模式。
- 提交适配：根据不同模式的表现，为最终课程大作业选择更可信的默认参数。

## 工程实现

`platform_modes.py` 会生成不同传播模式下的水印图和提取结果。每个模式代表一种使用场景，而不是随机攻击。这样输出结果更容易解释，也更方便在报告中说明为什么选择某一档参数。

## 结果解读

重点看同一水印配置在不同平台模式下的恢复稳定性。如果某个模式下画质尚可但恢复率明显更高，可以作为最终系统的推荐配置；如果某个模式只在极端参数下有效，就应当把它标为高风险或特殊场景。
