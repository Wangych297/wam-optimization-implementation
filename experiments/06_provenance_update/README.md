# 06 来源更新

## 目标

本实验面向图像再授权、再发布和版本更新场景。现实中一张图可能不是只发布一次：它可能先由原作者发布，再被平台处理、机构转发或二次授权。此时需要考虑新旧来源信息能否在同一张图中共存。

## 创新设计

来源更新的核心创新是把水印从“一次写入的静态标记”扩展为“可追加的来源记录”。实验会在已有水印图上再次写入新的来源消息，然后分别检查旧消息和新消息的可恢复情况。

这个设计关注三个问题：

- 新水印是否会覆盖或破坏旧水印。
- 旧水印是否仍能作为原始来源证据。
- 新水印是否能表达再发布、再授权或版本更新信息。

## 工程实现

`pipeline.py` 会执行两阶段流程：先生成初始水印图，再在该图上追加新的来源信息。随后对两个阶段的水印消息进行提取和对比，记录图像质量变化和消息恢复情况。

## 结果解读

理想情况是新旧消息都能保持一定可恢复性，且图像质量没有明显崩坏。如果新消息恢复较好但旧消息丢失严重，说明追加策略更像覆盖写入；如果旧消息保留较好但新消息不稳定，说明更新强度或区域安排不足。
